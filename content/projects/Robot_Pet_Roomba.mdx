## Robot Pet

Roomba 960, Raspberry Pi 5, Llama 3.1 70B, and zero monthly cost after hardware.

---

## Overview

A Roomba 960 converted into a voice controlled AI companion. It wanders the house, sees through a webcam, hears you speak, and responds with personality. Audio and video stay inside the house. Only transcribed conversation text is sent to the cloud.

The goal was to build something more physical and more playful than a chatbot on a laptop. This project combines speech, vision, memory, and movement into a single system built from relatively accessible hardware.

---

## Conversation Loop

You speak, whisper.cpp transcribes locally, Groq runs the reasoning step using text only, Piper speaks the response locally, and Python sends movement commands to the Roomba over serial.

Moondream adds room awareness by turning webcam images into short descriptions that can be passed into the prompt as context.

| Data or Function | Where it runs |
|---|---|

| Conversation memory | Local |
| LLM inference on transcribed text | Cloud |

## Hardware

| Component | Details |
|---|---|
| Base robot | Roomba 960 with vSLAM, Open Interface serial port, and WiFi |
| Main computer | Raspberry Pi 5 with 8GB RAM |
| Storage | 256GB or larger M.2 NVMe SSD with PCIe HAT |
| Serial adapter | 3.3V FTDI USB serial adapter |
| Camera | 1080p USB webcam such as a Logitech C270 or C920 |
| Audio | USB speakerphone such as the Anker PowerConf S330 |
| Power | USB C power bank with 25W PD output |
| Safety | Emergency stop button on GPIO pin 17 |

The 3.3V serial adapter is critical. A 5V adapter can damage the Roomba OI board.

---

## Software Stack

| Layer | Technology |
|---|---|
| Hearing | whisper.cpp for offline speech to text |
| Vision | Moondream 0.5B for webcam scene descriptions |
| Reasoning | Groq running Llama 3.1 70B |
| Speech | Piper TTS for offline voice output |
| Motion | pyserial controlling the Roomba OI at 115200 baud |
| Memory | SQLite for conversation history, learned facts, and reminders |
| Control | Python state machine running as a systemd service |

---

## Build Phases

1. Verify the Roomba by running a full stock cleaning cycle and confirming navigation works normally.
2. Set up the Pi 5 with NVMe boot, SSH, and the serial connection to the Roomba.
3. Build the voice loop so whisper hears, Groq responds, and Piper speaks.
4. Add vision so Moondream can describe the room and provide context.
5. Parse action tags from the model response and translate them into movement commands.
6. Store conversations and learned facts in SQLite so the robot can retain memory across sessions.
7. Mount the hardware on the Roomba and run the system untethered.
8. Tune the robot personality, name, and response style.

---

## Why This Build Matters

This project was a practical way to learn how local AI, cloud inference, physical hardware, and autonomous behavior fit together. It also showed that a fun home robot can be built without expensive robotics hardware or ongoing subscription costs.
