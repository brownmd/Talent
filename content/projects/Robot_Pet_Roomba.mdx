## Robot Pet v3 — Hybrid Design

Roomba 960 · Pi 5 · Hybrid local+cloud · Pure Python · $0/month

---

## Overview

Hybrid architecture — private data stays local, only conversation text goes to Groq. Result: Llama 3.1 70B intelligence at 500+ tok/s for $0/month.

**The data flow — one conversation loop**

You speak → whisper.cpp hears (local) → Groq API thinks (cloud, text only) → Groq responds → Piper TTS speaks (local) + pyserial moves robot (local)

Moondream sees room (local) → description sent to Groq as context

| Component | Details |
|---|---|
| Base robot | Roomba 960 · $75 · vSLAM navigation · Full OI serial port · WiFi |
| Brain | Raspberry Pi 5 · 8GB · Runs audio, vision, speech, Python loop |
| Cloud brain (free) | Groq · Llama 3.1 70B · 500+ tok/s · 6,000 req/day free |
| Orchestration | Pure Python · ~300 lines · No frameworks |

**What stays local vs what goes to cloud**

| Data | Where |
|---|---|
| Your voice audio | Always local |
| Webcam video / images | Always local |
| Robot movement commands | Always local |
| Memory file | Always local |
| Transcribed text of what you said | Groq (text only) |
| Moondream room description (text) | Groq (text only) |

## Hardware

Pi 5 no longer needs to run a local LLM — less thermal stress, faster boot, more RAM free for audio and vision processing.

**01 · Roomba 960 · $75**
The base. vSLAM navigation, full OI serial port, WiFi. Run one full stock cleaning cycle before any modifications.

**02 · Raspberry Pi 5 · 8GB · ~$80**
Required for simultaneous whisper + Moondream + Python loop. Do not use Pi 4 — not enough RAM once Moondream is loaded.

**03 · M.2 NVMe SSD + PCIe HAT · ~$35**
256GB+. Moondream (1.8GB) and whisper models load from NVMe in seconds. Pimoroni NVMe Base or Pineberry HatDrive recommended.

**04 · 3.3V FTDI USB serial adapter · ~$12**
CRITICAL: must be 3.3V logic. The Roomba OI runs at 3.3V — a 5V adapter damages the OI board.

**05 · 7-pin Mini-DIN cable · ~$10**
Plugs into Roomba service port under the handle flap. Route through handle arch, not around outside.

**06 · USB-C power bank · 25W PD min · ~$45**
Pi 5 draws up to 5A under load. Must be a verified PD chip at 25W+. Anker 737 confirmed.

**07 · 1080p USB webcam · ~$30**
Forward-facing, 110°+ FOV. Logitech C270 or C920. Tilted slightly down. Mounted on front lip with velcro.

**08 · USB conference speaker/mic · ~$35**
Single USB device for both in and out. Anker PowerConf S330 or similar. High-gain mic for wake word detection.

**09 · Momentary push button · ~$3**
Physical emergency stop. Wired to Pi GPIO. Sends Roomba OI STOP byte (173) instantly. Non-negotiable.

**10 · Heavy-duty velcro + cable ties · ~$8**
Stack: power bank center-over-axle → Pi 5 case on top → webcam front lip.

Total one-time cost: ~$333. Monthly cost after: ~$1.50 electricity. No subscriptions. No API bills.

## Build Order

Build in strict phase order. Each phase must work independently before adding the next.

**Phase 1 · Verify the Roomba**
Run the 960 stock for one full cleaning cycle. Confirm it docks and navigates without error codes.

**Phase 2 · Flash and boot Pi 5**
Flash Pi OS Lite 64-bit to NVMe. Boot, SSH in, `sudo apt update && sudo apt upgrade -y`. Confirm NVMe is boot device.

**Phase 3 · Serial connection**
Wire 3.3V FTDI → Mini-DIN → Roomba service port. 10 lines of pyserial. Send START (128) + SAFE (131) + DRIVE. Roomba moves. First milestone.

**Phase 4 · Groq voice loop**
whisper hears you → Groq responds as pet personality → Piper speaks it back. Full voice conversation before any robot movement.

**Phase 5 · Add vision**
Install Moondream. Add look() function. Wire into Groq prompt: "You can see: {room_description}."

**Phase 6 · Action parsing**
Groq appends action tags: [DRIVE_FORWARD], [SPIN_LEFT], [DOCK], [NONE]. Parse in Python, execute matching serial command.

**Phase 7 · SQLite memory**
Store every conversation turn. Retrieve 5 most relevant past memories before each Groq call. Robot remembers you across sessions.

**Phase 8 · State machine + idle behaviors**
Wrap in three-state loop: idle roam, listening, proactive. Add bump reaction, low battery dock, morning check-in.

**Phase 9 · Physical mount + systemd**
Mount hardware on Roomba. Register Python script as systemd service. Add GPIO kill switch. First untethered run.

**Phase 10 · Personality tuning**
Iterate on the Groq system prompt. Name, backstory, speech patterns, emotional responses. This is what makes it feel alive.

## Security

The hybrid model reduces security risk significantly. Attack surface: one API key, one serial port, one Python process.

**Protect the Groq API key**
Store in `/etc/environment` or `.env` with `chmod 600`. Never hardcode in Python. Never commit to GitHub.

**Hard-cap all serial commands**
Maintain an explicit allowlist: `DRIVE_FORWARD`, `SPIN_LEFT`, `SPIN_RIGHT`, `STOP`, `DOCK`. Anything else is silently dropped. Speed hard-capped at 200mm/s. The LLM never touches serial directly.

**GPIO emergency stop button**
Wire momentary button to GPIO pin 17. When pressed: send STOP byte (173), kill drive thread, log event. Costs $3. Non-negotiable.

**SSH key auth only**
Set `PasswordAuthentication no` in `/etc/ssh/sshd_config`. Copy your SSH public key first with `ssh-copy-id`.

**Robot on IoT VLAN**
Dedicated guest/IoT WiFi with client isolation. Robot reaches Groq API but cannot reach your laptop, phone, or NAS.

**ufw firewall**
`sudo ufw default deny outgoing` then allow: NTP (123), Groq (api.groq.com:443), SSH (22). Block everything else.

Groq never receives: your voice audio · webcam images · network details · location · robot sensor data. Only transcribed conversation text.
