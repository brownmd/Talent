  <div class="step-row"><div class="step-num">3</div><div class="step-body"><div class="step-title">Phase 3 · Serial connection</div><div class="step-desc">Wire 3.3V FTDI → Mini-DIN → Roomba service port. 10 lines of pyserial.   <div class="step-row"><div class="step-num">4</div><div class="step-body"><div class="step-title">Phase 4 · Groq voice loop</div><div class="step-desc">whisper hears you → Groq responds as pet personality → Piper speaks it back. Full voice conversation before any robot movement.</div></div></div>
  <div class="step-row"><div class="step-num">5</div><div class="step-body"><div class="step-title">Phase 5 · Add vision</div><div class="step-desc">Install Moondream. Add look() function.   <div class="step-row"><div class="step-num">6</div><div class="step-body"><div class="step-title">Phase 6 · Action parsing</div><div class="step-desc">Groq appends action tags: [DRIVE_FORWARD], [SPIN_LEFT], [DOCK], [NONE].   <div class="step-row"><div class="step-num">7</div><div class="step-body"><div class="step-title">Phase 7 · SQLite memory</div><div class="step-desc">Store every conversation turn. Retrieve 5 most relevant past memories before each Groq call. Robot remembers you across sessions.</div></div></div>

<!-- SECURITY -->
<div id="security" class="panel">
  <div class="info">The hybrid model reduces security risk vs full OpenClaw significantly. Attack surface: one API key, one serial port, one Python process.</div>
  <div class="card"><div class="badge b-red">Top risk · API key</div><div class="card-title">Protect the Groq API key</div><div class="card-sub">Store in <span class="mono">/etc/environment</span> or <span class="mono">.env</span> with <span class="mono">chmod 600</span>. Never hardcode in Python. Never commit to GitHub. The key only allows text generation — it cannot access your network, files, or robot.</div></div>
  <div class="card"><div class="badge b-red">Top risk · Serial</div><div class="card-title">Hard-cap all serial commands</div><div class="card-sub">Maintain an explicit allowlist: <span class="mono">DRIVE_FORWARD</span>, <span class="mono">SPIN_LEFT</span>, <span class="mono">SPIN_RIGHT</span>, <span class="mono">STOP</span>, <span class="mono">DOCK</span>. Anything else is silently dropped. Speed hard-capped at 200mm/s. The LLM never touches serial directly.</div></div>
  <div class="card"><div class="badge b-amber">Hardware</div><div class="card-title">GPIO emergency stop button</div><div class="card-sub">Wire momentary button to GPIO pin 17. When pressed: send STOP byte (173), kill drive thread, log event. Costs $3. Non-negotiable safety hardware.</div></div>
  <div class="card"><div class="badge b-amber">SSH</div><div class="card-title">Key auth only, no passwords</div><div class="card-sub">Set <span class="mono">PasswordAuthentication no</span> in <span class="mono">/etc/ssh/sshd_config</span>. Copy your SSH public key first with <span class="mono">ssh-copy-id</span>.</div></div>
  <div class="card"><div class="badge b-amber">Network</div><div class="card-title">Robot on IoT VLAN</div><div class="card-sub">Dedicated guest/IoT WiFi with client isolation. Robot reaches Groq API but cannot reach your laptop, phone, or NAS.</div></div>
  <div class="card"><div class="badge b-amber">Firewall</div><div class="card-title">ufw — allow only what you need</div><div class="card-sub"><span class="mono">sudo ufw default deny outgoing</span> then allow: NTP (123), Groq (api.groq.com:443), SSH (22). Block everything else.</div></div>
  <div class="card"><div class="badge b-green">Privacy</div><div class="card-title">What Groq never receives</div><div class="card-sub">Your voice audio · webcam images · network details · location · robot sensor data. Only transcribed conversation text — same as a web search.</div></div>
  <div class="success">No Node.js attack surface · No gateway port · No skill injection · No cloud audio or video · No messaging channel to hijack. Security posture is genuinely good.</div>
</div>
