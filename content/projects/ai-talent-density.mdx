## Overview

This is a monthly updated analytics pipeline with production guardrails and stable consumer facing views. It is not a one off heatmap.

The current pipeline processes 5.87M GitHub activity records, loads monthly developer scores into Postgres/PostGIS, normalizes raw profile locations against a curated GeoNames reference, and serves downstream outputs to Metabase and Kepler.gl.

---

## What This Project Does

This repo turns noisy public GitHub activity into a structured AI talent dataset.

It does four things:

1. Identifies AI-relevant GitHub activity from GitHub Archive using curated repo-pattern filters.
2. Scores developers monthly based on event type and repo-category multipliers.
3. Enriches users with public GitHub profile locations and normalizes those locations into canonical city, region, country, and metro aware rollups.
4. Publishes analytics-friendly views for dashboards, reporting, and geographic visualization.

## Scoring Model

### Event weights

Higher signal contribution events receive more weight than passive activity.

Examples:

- ReleaseEvent: 30
- PullRequestReviewEvent: 25
- DeploymentEvent: 25
- PullRequestEvent: 20
- CreateEvent: 15
- CheckRunEvent: 15
- PushEvent: 10
- WorkflowRunEvent: 10
- MemberEvent: 10
- IssuesEvent: 5
- IssueCommentEvent: 4
- ForkEvent: 2
- WatchEvent: 1

### Repo multipliers

Repos are grouped into AI relevant ecosystems and weighted by technical sophistication.

Current multiplier bands:

- 5.0x: scaling, interpretability, RL, frontier infra
- 3.5x: agentic orchestration and MCP ecosystems
- 2.0x: SDKs, frameworks, and builder tools
- 1.5x: local tooling and AI assisted development tools
- 1.0x: all other qualifying AI activity
